# Implementation Plan: Hub Shard Display

**Branch**: `017-hub-shard-display` | **Date**: 2026-03-01 | **Spec**: [spec.md](spec.md)

## Summary

Add a screen-space shard counter to the hub. A `ShardDisplay` component script is attached to a
`Control` node inside a `CanvasLayer` child of `HubRoom.tscn`. On `_ready()` it reads
`MetaManager.meta_state.total_shards` and sets a `Label`'s text to `"Shards: N"`. The overlay
disappears automatically when HubRoom is freed (run starts), satisfying the hide-during-runs
requirement with no extra wiring.

## Technical Context

**Language/Version**: GDScript (Godot 4.6), static typing throughout
**Primary Dependencies**: `MetaManager` autoload (already implemented in feature 016)
**Storage**: None — read-only display
**Testing**: Manual validation scenarios in `quickstart.md`
**Target Platform**: Android mobile (portrait 1080×1920); Windows for development
**Project Type**: Mobile game (single project)
**Performance Goals**: None — single Label update in `_ready()`; no per-frame cost
**Constraints**: Must not intercept touch input on TeleportDoor; screen-space via CanvasLayer

## Implementation Notes

### ShardDisplay.gd

```gdscript
extends Control

@export var _label: Label

func _ready() -> void:
    _label.text = "Shards: {n}".format({"n": MetaManager.meta_state.total_shards})
```

### HubRoom.tscn changes (Godot Editor)

1. Add `CanvasLayer` child to `HubRoom` — name it `ShardDisplayLayer`.
2. Add `Control` child to `ShardDisplayLayer` — name it `ShardDisplay`. Attach `ShardDisplay.gd`.
3. Add `Label` child to `ShardDisplay` — name it `Label`. Position top-left or top-right.
4. In Inspector on `ShardDisplay`: assign `_label` → `Label`.
5. Set `Mouse Filter` to `Ignore` on both `ShardDisplay` and `Label` so touch passes through.
