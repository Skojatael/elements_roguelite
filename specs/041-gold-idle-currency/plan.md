# Implementation Plan: Gold Idle Currency

**Branch**: `041-gold-idle-currency` | **Date**: 2026-03-13 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/041-gold-idle-currency/spec.md`

---

## Summary

Add a persistent `gold` currency that accumulates at 100/hour in real time, including while the game is closed, using a timestamp-delta approach. On launch, offline credit is applied before the hub is shown. While in the hub, the gold label visibly increments in real time. No player action is required at any point. Gold is display-only in this feature. The implementation extends MetaState, SaveManagerImpl, MetaManagerImpl, and the MetaManager autoload with minimal surface area, then adds a new GoldDisplay scene in the hub.

**Scale/Scope**: 5 modified files + 1 new scene (GoldDisplay)

## Constitution Check

### I. Single Responsibility ✅
- `GoldDisplay.gd` owns only the gold label — no accumulation logic.
- `MetaManagerImpl` gains two focused methods (`tick_gold`, `apply_offline_gold`); no mixed concerns.
- `MetaManager` autoload remains a thin wrapper: new `_process()` is a 4-line delegation + signal emit.
- No new autoload introduced.

### II. Data-Driven Content ✅
- `gold_rate_per_hour: 100` lives in `data/meta_config.json`.
- No hardcoded rate constants in any `.gd` file.

### III. Mobile-First Performance ✅
- `_process()` does one float add and one `floori()` comparison per frame — negligible.
- `gold_changed` signal emits at most once per second (only when floor changes at 100/hr rate).
- Label text is set via signal, not polled in `_process()`.

### IV. Editor-Centric Workflow ✅
- `GoldDisplay.tscn` created in Godot Editor; `.tscn` not edited as raw text.
- `_label: Label` is `@export var` assigned via Inspector.
- No hardcoded `$NodeName` paths outside of architecturally fixed subtrees.

### V. Simplicity & YAGNI ✅
- No shared base class between `GoldDisplay` and `ShardDisplay` — two scenes is not enough to justify abstraction.
- No spending mechanic stub; no upgrade hook; no cap hook. Deferred to future features.
- `_save()` helper is refactor of existing callsites, not a new abstraction layer.

### VI. Early Return ✅
- `apply_offline_gold`: two guard clauses at top (timestamp == 0, elapsed <= 0), happy path unindented.
- `_process()`: early return when floor unchanged.
- All new methods exit at first unmet precondition.

**Verdict**: All principles satisfied. Proceeding to Phase 1.
