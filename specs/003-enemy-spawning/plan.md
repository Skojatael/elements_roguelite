# Implementation Plan: Enemy Spawning

**Date**: 2026-02-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-enemy-spawning/spec.md`

---

## Summary

When the player enters a dungeon room, enemies defined in `dungeon_config.json` are instantiated at configured positions (with optional randomisation radius) and added to the scene. The room tracks living enemy count; when it reaches zero the room is flagged as cleared in `RunManager` and no enemies re-spawn on re-entry.

---

## Technical Context

**Language/Version**: GDScript (Godot 4.6)
**Primary Dependencies**: Godot 4.6 engine; Jolt physics; `Enemy.tscn` from `002-enemy-combat`
**Storage**: `data/dungeon_config.json` (extended); run-time cleared state in `RunManager` autoload (in-memory Dictionary)
**Testing**: Manual playtest per `quickstart.md`; Godot output panel for error validation
**Target Platform**: Android mobile (portrait 1080×1920); Windows for development
**Performance Goals**: ≤10 enemies per room spawn simultaneously with no visible frame-rate drop on mid-range Android (SC-005)
**Constraints**: All enemies must appear within 0.5 s of room entry (SC-001); cleared state must update within one frame of last defeat (SC-003)
**Scale/Scope**: Up to 10 enemies per room; 4 room types in initial dungeon set

## Constitution Check

*Gate: Must pass before Phase 0 research. Re-checked after Phase 1 design.*

### I. Single Responsibility ✅

- `RoomSpawner.gd` owns exactly one concern: spawn, track, clear.
- `SpawnPointData.gd` and `RoomSpawnConfig.gd` own only JSON-to-typed-object parsing.
- `RunManager` extension adds `cleared_rooms` to its existing run-state domain — no new domain created.
- Each script is co-located with its scene family (`scenes/dungeon/`) or in `scripts/data_models/` as a shared model.

### II. Data-Driven Content ✅

- All spawn configuration (enemy IDs, positions, radii) lives in `data/dungeon_config.json`.
- GDScript data models (`SpawnPointData`, `RoomSpawnConfig`) provide typed wrappers; `RoomSpawner` never reads raw JSON dictionaries.
- Changing enemy types or quantities requires only editing the JSON file — zero code changes (SC-002).

### III. Mobile-First Performance ✅

- Maximum 10 enemies per room enforced at data load time (FR-009).
- Room entry detection uses `Area2D.body_entered` — zero-cost at idle, one-shot signal.
- Enemy instantiation is synchronous but bounded to ≤10 calls per entry event; no async loading needed.
- No new shaders or materials introduced.

### IV. Editor-Centric Workflow ✅

- Room scene modifications (adding `EntryArea` + `RoomSpawner` nodes) are made exclusively in the Godot Editor.
- `.tscn` files are never edited as raw text.
- `room_id` export var is set per room instance in the Inspector.

### V. Simplicity & YAGNI ✅

- No wave system, no spawn animations, no timed delays — immediate instantiation only.
- `RoomSpawnConfig` and `SpawnPointData` wrappers are introduced at exactly two call sites (`RoomSpawner` + data validation), satisfying the two-site threshold.
- No future hooks or placeholder infrastructure.

### SpawnPointData.gd

```gdscript
class_name SpawnPointData
extends RefCounted

var enemy_id: String = ""
var position: Vector2 = Vector2.ZERO
var radius: float = 0.0

static func from_dict(d: Dictionary) -> SpawnPointData:
    var s := SpawnPointData.new()
    s.enemy_id = d.get("enemy_id", "")
    var pos: Dictionary = d.get("position", {})
    s.position = Vector2(pos.get("x", 0.0), pos.get("y", 0.0))
    s.radius = float(d.get("radius", 0.0))
    return s
```

---

### RoomSpawnConfig.gd

```gdscript
class_name RoomSpawnConfig
extends RefCounted

var room_id: String = ""
var spawn_points: Array[SpawnPointData] = []

static func from_dict(room_id: String, d: Dictionary) -> RoomSpawnConfig:
    var cfg := RoomSpawnConfig.new()
    cfg.room_id = room_id
    for entry in d.get("spawn_points", []):
        cfg.spawn_points.append(SpawnPointData.from_dict(entry))
    return cfg
```

### RoomSpawner.gd (full logic outline)

```gdscript
class_name RoomSpawner
extends Node

const ENEMY_SCENE := preload("res://scenes/combat/enemies/Enemy.tscn")
const MAX_ENEMIES := 10

@export var room_id: String = ""

signal room_cleared

@onready var _entry_area: Area2D = $EntryArea

var _config: RoomSpawnConfig
var _living_count: int = 0
var _spawned: bool = false

func _ready() -> void:
    _config = _load_config()
    _entry_area.body_entered.connect(_on_player_entered)

func _load_config() -> RoomSpawnConfig:
    var raw: Dictionary = ResourceManager.get_dungeon_config()
    var configs: Dictionary = raw.get("spawn_configs", {})
    if not configs.has(room_id):
        return RoomSpawnConfig.new()           # empty = no spawns, no error
    var cfg := RoomSpawnConfig.from_dict(room_id, configs[room_id])
    # Validate count
    if cfg.spawn_points.size() > MAX_ENEMIES:
        push_error("RoomSpawner: spawn_points count exceeds maximum of 10 in room '%s'" % room_id)
        return RoomSpawnConfig.new()
    # Validate enemy IDs
    for sp in cfg.spawn_points:
        if not ResourceManager.enemy_id_exists(sp.enemy_id):
            push_error("RoomSpawner: unknown enemy_id '%s' in room '%s'" % [sp.enemy_id, room_id])
            return RoomSpawnConfig.new()
    return cfg

func _on_player_entered(body: Node2D) -> void:
    if not (body is CharacterBody2D and body.is_in_group("player")):
        return
    if RunManager.is_room_cleared(room_id):
        return
    if _spawned:
        return
    _spawn_enemies()

func _spawn_enemies() -> void:
    _spawned = true
    _living_count = _config.spawn_points.size()
    if _living_count == 0:
        return
    for sp in _config.spawn_points:
        var enemy: Enemy = ENEMY_SCENE.instantiate()
        enemy.enemy_type_id = sp.enemy_id
        get_parent().add_child(enemy)
        var offset := Vector2(
            randf_range(-sp.radius, sp.radius),
            randf_range(-sp.radius, sp.radius)
        )
        enemy.global_position = sp.position + offset   # must be after add_child
        enemy.defeated.connect(_on_enemy_defeated)

func _on_enemy_defeated() -> void:
    _living_count -= 1
    if _living_count == 0:
        RunManager.mark_room_cleared(room_id)
        room_cleared.emit()
```

### RunManager additions

```gdscript
var cleared_rooms: Dictionary = {}

func mark_room_cleared(room_id: String) -> void:
    cleared_rooms[room_id] = true

func is_room_cleared(room_id: String) -> bool:
    return cleared_rooms.has(room_id)
```

Reset `cleared_rooms = {}` in the existing `start_new_run()` method (or equivalent).

### Editor tasks (per room scene — all four room scenes)

In the Godot Editor, for each of `CombatRoom01.tscn`, `CombatRoom02.tscn`, `EliteRoom01.tscn`, `BossRoom01.tscn`:

1. Add child `Area2D` named `EntryArea`.
2. Add `CollisionShape2D` child to `EntryArea`; assign a `RectangleShape2D` sized to the room floor.
3. Set `EntryArea` collision: Layer = 0 (none), Mask = 1 (detects player body Layer 1).
4. Add child `Node` named `RoomSpawner`; attach `res://scenes/dungeon/RoomSpawner.gd`.
5. Set `room_id` export to the room's type string (e.g. `"CombatRoom01"`).

## Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| `ResourceManager` does not yet expose `get_dungeon_config()` | Implement helper in `ResourceManager` as part of Setup phase; it's a stub today |
| `enemy_id_exists()` requires loading `enemies.json` again | `ResourceManager` can cache both configs; or `RoomSpawner` loads `enemies.json` directly as a one-time dict |
| Player may not be in `"player"` group | Add `player` group to Player node in editor during Setup phase |
| `Enemy.global_position` must be set after `add_child` if the scene uses `@onready` nodes | Set `global_position` after `add_child`; `enemy_type_id` is set before `add_child` (safe because `_ready` runs at `add_child` time) |
