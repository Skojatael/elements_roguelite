# Implementation Plan: Player State Snapshot

**Branch**: `012-player-state` | **Date**: 2026-02-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/012-player-state/spec.md`

## Summary

Add a `PlayerState` RefCounted data class that holds the player's current HP (synced live via `StatsComponent.health_changed`) plus four stub fields (`items`, `modifiers`, `skill_changes`, `skill_cooldowns`). `RunManager` owns the sole write reference and resets `PlayerState` at `end_run()`. `RunState` gains a `player_state` property so all consumers access the snapshot through a single path.

## Technical Context

**Language/Version**: GDScript 4 (Godot 4.6)
**Primary Dependencies**: Godot 4.6 engine — no third-party libraries
**Storage**: Runtime only — no disk persistence in this feature
**Testing**: Manual validation via Godot Remote Inspector and Output panel (see quickstart.md)
**Target Platform**: Mobile (Android/iOS); Windows editor
**Project Type**: Single Godot project
**Performance Goals**: Zero per-frame cost — health sync is signal-driven (event-based)
**Constraints**: `PlayerState` must never be null; `current_hp` must equal `StatsComponent.current_health` at all times during a run
**Scale/Scope**: 1 new file, 2 modified files, 1 new method, 4 signal connection lines

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **SRP** ✅ — `PlayerState` is a single-responsibility data class. `RunManager` remains sole writer; all consumers are read-only.
- **One file, one class** ✅ — `PlayerState.gd` contains only `PlayerState`.
- **RefCounted for runtime data** ✅ — Matches `RunState` and `SpawnContext` patterns. No disk serialization required.
- **Signal-driven updates** ✅ — Reuses existing `StatsComponent.health_changed`; no polling.
- **No new signals** ✅ — Consumers poll `RunManager.run_state.player_state`. No new signals introduced.
- **Guard against duplicate connections** ✅ — `is_connected()` check before `stats.health_changed.connect(...)` in `start_run()`.

*Post-design re-check*: All constraints satisfied. No violations.

## Implementation Phases

### Phase 1 — PlayerState.gd (new file)

**File**: `scripts/data_models/PlayerState.gd`

```gdscript
class_name PlayerState
extends RefCounted

## Read-only for all systems except RunManager.

## Player's current health. Synced from StatsComponent.health_changed.
var current_hp: float = 0.0

## Stub: items acquired this run. Always empty until item system is implemented.
var items: Array = []

## Stub: active stat modifiers. Always empty until modifier system is implemented.
var modifiers: Array = []

## Stub: run-specific skill modifications. Always empty until skill-change system is implemented.
var skill_changes: Array = []

## Stub: per-skill cooldown state (skill_id → remaining_cooldown). Always empty until cooldown tracking is implemented.
var skill_cooldowns: Dictionary = {}
```

### Phase 2 — RunState extension

**File**: `scripts/data_models/RunState.gd`

Add after the last existing field:

```gdscript
## Player state snapshot for this run. Same reference as RunManager.player_state.
var player_state: PlayerState = PlayerState.new()
```

Initialized at declaration → safe defaults (current_hp = 0.0, stubs empty) before any run.

### Phase 3 — RunManager: new field

**File**: `autoload/RunManager.gd`

Add at the top-level variable block (after existing vars):

```gdscript
## Player state snapshot. Updated via health_changed signal; reset in end_run().
var player_state: PlayerState = PlayerState.new()
```

### Phase 4 — RunManager: start_run() setup

**File**: `autoload/RunManager.gd`

After the existing `stats.reset()` call block in `start_run()`, add:

```gdscript
player_state = PlayerState.new()
run_state.player_state = player_state

var players: Array = get_tree().get_nodes_in_group("player")
for player: Node in players:
    var stats: StatsComponent = player.get_node_or_null("StatsComponent")
    if stats != null:
        if not stats.health_changed.is_connected(_on_player_health_changed):
            stats.health_changed.connect(_on_player_health_changed)
        player_state.current_hp = stats.current_health
```

Note: `stats.reset()` (already called before this block) fires `health_changed` → `_on_player_health_changed()` runs → `player_state.current_hp` is set to `max_health` automatically. The explicit `player_state.current_hp = stats.current_health` line handles the window before reset fires.

### Phase 5 — RunManager: end_run() reset

**File**: `autoload/RunManager.gd`

After `is_run_active = false` in `end_run()`, add:

```gdscript
var players: Array = get_tree().get_nodes_in_group("player")
for player: Node in players:
    var stats: StatsComponent = player.get_node_or_null("StatsComponent")
    if stats != null:
        player_state = PlayerState.new()
        player_state.current_hp = stats.max_health
        run_state.player_state = player_state
        break
```

Reset creates a fresh instance (stubs empty, current_hp = max_health). Both `player_state` and `run_state.player_state` updated to keep them in sync.

### Phase 6 — RunManager: _on_player_health_changed()

**File**: `autoload/RunManager.gd`

Add new method:

```gdscript
func _on_player_health_changed(new_health: float, _max_health: float) -> void:
    player_state.current_hp = new_health
```

Underscore prefix on `_max_health` — only `new_health` is used.

### Phase 7 — Validation

Run all 11 scenarios in `quickstart.md` using Godot Remote Inspector and Output panel.

## State Lifecycle

```
Before any run:
  player_state = PlayerState.new()          ← declared at RunManager init
  player_state.current_hp = 0.0            ← default
  run_state.player_state = PlayerState.new() ← declared at RunState init

start_run(mode):
  player_state = PlayerState.new()
  run_state.player_state = player_state     ← shared reference established
  Connect stats.health_changed → _on_player_health_changed (if not connected)
  player_state.current_hp = stats.current_health
  stats.reset() → health_changed fires → player_state.current_hp = stats.max_health

During run:
  take_damage() → health_changed → player_state.current_hp updates
  heal()        → health_changed → player_state.current_hp updates

end_run():
  player_state = PlayerState.new()           ← fresh instance
  player_state.current_hp = stats.max_health ← full HP
  run_state.player_state = player_state      ← both references updated
  is_run_active = false
```
