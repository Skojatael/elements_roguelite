# Quickstart: Room Factory

**Feature**: 006-room-factory
**Date**: 2026-02-23
**Purpose**: Manual validation scenarios for all functional requirements.

---

## Prerequisites

- Godot 4.6 editor open with `project.godot`
- `004-run-manager` complete — `RunManager.start_run()` and `end_run()` functional
- `005-dev-panel` complete — dev panel available for run control
- `Main.tscn` is the active main scene

---

## Scenario 1 — Factory spawns a known room type (FR-001, FR-002, FR-003, FR-004)

**Steps**:
1. In a temporary test script (or GDScript console), call:
   ```gdscript
   var room_data: RoomData = load("res://data/rooms/CombatRoom01.tres")
   var context := SpawnContext.create(get_tree().root, Vector2(0, 0))
   var controller := RunManager.spawn_room(room_data, "test_room_01", context)
   ```
2. Run the game.

**Expected**:
- No errors in Output.
- A `CombatRoom01` scene appears at world position `(0, 0)` as a child of the scene root.
- `controller` is not null.
- `controller.room_id` equals `"test_room_01"`.
- Output: `[RoomFactory] spawned room type='CombatRoom01' id='test_room_01'`

**Pass criterion**: RoomController returned, room visible in world at correct position.

## Scenario 2 — Null or invalid RoomData returns null, no crash (FR-008)

**Steps**:
1. Call `RunManager.spawn_room(null, "test_001", context)`.

**Expected**:
- Output / Debugger: error message mentioning unknown room type.
- Return value is `null`.
- No scene node added to the tree.
- Game does not crash.

**Pass criterion**: null returned, error logged, scene tree unchanged.

---

## Scenario 3 — Factory-created room does NOT auto-register with RunManager (Decision 4)

**Steps**:
1. Set a breakpoint or add a print in `RunManager.register_room()`.
2. Spawn a room via `RunManager.spawn_room(...)`.

**Expected**:
- `register_room()` is NOT called for the factory-created room.
- RoomSpawner's `auto_register` is `false` for factory rooms.

**Pass criterion**: `register_room()` print does not fire for factory-created rooms.

---

## Scenario 4 — RoomController emits room_entered on player entry (FR-006)

**Steps**:
1. Spawn `"CombatRoom01"` at a position the player can walk into.
2. Connect to `controller.room_entered` with a print callback.
3. Walk the player into the room's EntryArea.

**Expected**:
- Output: callback fires with `room_id = "test_room_01"`.
- RunManager's `_on_room_entered` also fires (because RunManager connected to the controller).

**Pass criterion**: `room_entered` signal fires exactly once on entry.

---

## Scenario 5 — RoomController emits room_cleared after all enemies defeated (FR-007)

**Steps**:
1. Using the room from Scenario 4, defeat all spawned enemies.

**Expected**:
- Output: `[RunManager] room cleared 'test_room_01'`
- `RunManager.is_room_cleared("test_room_01")` returns `true`.

**Pass criterion**: `room_cleared` fires once all enemies are defeated.

---

## Scenario 6 — Pre-placed rooms continue to work (backward compatibility)

**Steps**:
1. Leave existing pre-placed rooms in `Main.tscn` unchanged.
2. Run the game normally.

**Expected**:
- Pre-placed rooms behave identically to before this feature.
- `register_room()` is called for pre-placed rooms (auto_register=true).
- No errors related to RoomController or RoomFactory.

**Pass criterion**: No regression in pre-placed room behaviour.

---

## Scenario 7 — room_id is never generated by factory or RunManager (FR-010)

**Steps**:
1. Inspect `RoomFactory.spawn_room()` and `RunManager.spawn_room()` source.

**Expected**:
- Neither function contains any ID generation logic (no `str(Time.get_ticks_msec())`,
  no UUID, no counter).
- The `room_id` parameter is passed through as-is.

**Pass criterion**: Code review confirms zero ID generation in factory or RunManager.
