# Implementation Plan: Room Factory

**Date**: 2026-02-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/006-room-factory/spec.md`

---

## Summary

A `RoomFactory` (RefCounted, owned by RunManager) receives `(room_data: RoomData, room_id, context)`
and returns a `RoomController` — a thin signal-forwarding wrapper. `RoomData` is a Godot `Resource`
saved as `.tres` files in `res://data/rooms/`; it carries the room scene reference directly, so
RoomFactory needs no internal scene registry. The factory reads `room_data.scene`, instantiates it,
sets room_id and auto_register=false on the RoomSpawner before add_child, positions the room, and
returns the controller. RunManager gains a `spawn_room()` method and updates `register_room()` to
also produce a RoomController, making `current_room` uniformly typed.

Room IDs are never generated by RoomFactory or RunManager — always caller-supplied (FR-010).

---

## Technical Context

**Language/Version**: GDScript (Godot 4.6)
**Primary Dependencies**: Godot 4.6 scene system; `RunManager` autoload (004); `RoomSpawner` (003)
**Storage**: None — factory is stateless; rooms live in the scene tree
**Testing**: Manual playtest per `quickstart.md`; Godot Output panel for log validation
**Target Platform**: Android mobile (portrait 1080×1920); Windows for development
**Performance Goals**: No overhead beyond normal scene instantiation; RoomController is a RefCounted
with two signal connections — negligible
**Constraints**: room_id always caller-supplied; factory must be stateless; no new autoload
**Scale/Scope**: 3 new files, 2 modified files

## Constitution Check

### I. Single Responsibility ✅

- `RoomFactory` — one responsibility: instantiate a room scene and return a RoomSpawner.
- `SpawnContext` — one responsibility: bundle parent node + position for a spawn call.
- `RunManager` — existing orchestrator; `spawn_room()` is a natural extension of its run-lifecycle
  domain. Factory is a dependency, not merged logic.
- No new autoload. Factory is owned by RunManager as an instance var.

### II. Data-Driven Content ✅

- No balance values. Room type → scene mapping is infrastructure, not tuneable content.

### III. Mobile-First Performance ✅

- `RoomController extends RefCounted` — zero scene-tree overhead.
- `SpawnContext extends RefCounted` — zero scene-tree overhead.
- Factory uses `preload()` — no runtime I/O on spawn call.

### IV. Editor-Centric Workflow ✅

- No `.tscn` files modified as text. RoomSpawner's new `auto_register` field is an `@export`
  visible in the Editor Inspector.

### V. Simplicity & YAGNI ✅

- `RoomData` collapsed to a `String` parameter (one field, one consumer — class unjustified).
- Factory is a plain `RefCounted`, not an autoload.
- `auto_register` flag is the minimal change to RoomSpawner for backward compat.

## Phase 0: Research Summary

All decisions resolved. See `research.md` for full rationale.

| Decision | Choice |
|---|---|
| Room type → scene mapping | `RoomData extends Resource` (`.tres` per room type in `res://data/rooms/`) |
| Return type | `RoomSpawner` directly — no wrapper needed (YAGNI) |
| SpawnContext | `RefCounted` data class in `scripts/data_models/` |
| Auto-registration conflict | `auto_register: bool = true` on RoomSpawner; factory sets `false` |
| RoomData | `Resource` with `@export scene: PackedScene` + `@export room_type_id: String` |
| Factory ownership | Owned by RunManager as instance var; not an autoload |

## Notes

- `RoomSpawner.auto_register` is an `@export` so existing pre-placed rooms in the Editor keep
  the default `true` without any scene file changes.
- `RunManager.register_room()` return type changes from `void` to `RoomController` — this is
  backward compatible; existing callers (RoomSpawner) ignore the return value.
- `RunManager._on_room_entered` signature changes: currently `(room_id: String, spawner: Node)`;
  becomes `(room_id: String, controller: RoomController)`. The `.bind(controller)` call in
  `_connect_room_controller` handles this.
- `current_room` changes from `Node` to `RoomController` — update the CLAUDE.md field type table.
