# Implementation Plan: Dungeon Expansion (Adventuring Gear)

**Branch**: `033-dungeon-expansion` | **Date**: 2026-03-06 | **Spec**: specs/033-dungeon-expansion/spec.md

## Summary

Two meta-progression additions: (1) first boss kill sets `first_boss_killed` flag, which makes an "Adventuring Gear" purchase button visible in hub (300 shards); (2) purchasing the gear sets `adventuring_gear_owned`, causing every subsequent run's dungeon to expand by 4 rooms branching from the deepest base room (Room A). Base room count increased from 8 to 9 (8 combat rooms + 1 start room). Grid expanded from 5×5 to 13×13 to guarantee 4 expansion rooms always fit, proven by the Manhattan distance argument (see research.md).

## Technical Context

**Language/Version**: GDScript, Godot 4.6
**Primary Dependencies**: `MetaManagerImpl` (new methods), `MetaManager` (autoload wrapper), `DungeonGenerator` (expansion algorithm + grid resize), `SaveManagerImpl` (new fields), `AdventuringGearShop` (new scene)
**Storage**: `data/meta_config.json` (cost), `data/dungeon_config.json` (base_room_count, expansion_room_count), `user://meta_save.json` (new boolean fields)
**Testing**: Manual (quickstart.md, 10 scenarios)
**Target Platform**: Android mobile (portrait); Windows dev
**Project Type**: Single Godot project
**Performance Goals**: All computation same-frame; grid is 11×11 virtual (no scene overhead for unused cells)
**Constraints**: No new autoloads; no new JSON schemas beyond existing files
**Scale/Scope**: 2 data files modified, 5 script files modified, 1 new scene

## Constitution Check

- **I. Single Responsibility** ✅ `DungeonGenerator` remains the sole dungeon layout owner; `_expand_dungeon()` is a new private method within it, not a new class. `MetaManagerImpl` gains 2 pure methods. `AdventuringGearShop` is a single-responsibility hub UI component with one concern (show/hide + delegate purchase).
- **II. Data-Driven Content** ✅ `adventuring_gear_cost: 300` in `meta_config.json`; `expansion_room_count: 4` in `dungeon_config.json`. `GRID_SIZE` is an architectural constant (determines room ID namespace and coordinate system — changing it would invalidate save-file room IDs); kept as code const, justified.
- **III. Mobile-First** ✅ Dungeon generation is same-frame. 11×11 virtual grid has no scene overhead; unused cells cost nothing. No rendering changes.
- **IV. Editor-Centric** ✅ `AdventuringGearShop.tscn` created via Godot Editor. `@export var _button: Button` assigned in Inspector. No raw `.tscn` editing.
- **V. Simplicity & YAGNI** ✅ Frontier expansion reuses `_record_room()`. Boss kill detection reuses the existing `room_cleared` signal. Purchase pattern mirrors `UpgradeShop`. No new abstractions introduced.
- **VI. Early Return** ✅ `record_boss_kill()` returns early if already set. `purchase_adventuring_gear()` returns early if can't spend. `_on_room_cleared()` returns early after boss branch.

### T4b — DungeonGenerator.gd: _expand_dungeon()

Replace the `_promote_elite_rooms()` call in `_generate()` so it runs after expansion when gear is owned:

```gdscript
_build_neighbours(occupied)

if MetaManager.is_adventuring_gear_owned:
    _expand_dungeon(occupied, pool, difficulty_scale)
    _build_neighbours(occupied)

```

This replaces the existing `_build_neighbours(occupied)` + `_promote_elite_rooms()` call sequence in `_generate()`.

New methods:

```gdscript
func _expand_dungeon(occupied: Dictionary, pool: Array, difficulty_scale: float) -> void:
    var expansion_count: int = ResourceManager.get_dungeon_config().get("expansion_room_count", 4)
    var max_depth: int = 0
    for room_data: Variant in rooms_by_id.values():
        var d: int = (room_data as Dictionary).get("depth", 0)
        if d > max_depth:
            max_depth = d
    var seed_id: String = ""
    for rid: String in rooms_by_id:
        if rooms_by_id[rid].get("depth", 0) == max_depth:
            seed_id = rid
            break
    var seed_cell: Vector2i = rooms_by_id[seed_id]["grid_pos"]
    var expansion_frontier: Array[Vector2i] = _get_expansion_neighbours(seed_cell, occupied, max_depth)
    var added: int = 0
    while added < expansion_count and not expansion_frontier.is_empty():
        var idx: int = randi() % expansion_frontier.size()
        var cell: Vector2i = expansion_frontier[idx]
        expansion_frontier.remove_at(idx)
        _record_room(cell, pool.pick_random(), occupied, expansion_frontier, difficulty_scale)
        # Prune frontier: keep only cells strictly deeper than Room A
        var pruned: Array[Vector2i] = []
        for c: Vector2i in expansion_frontier:
            if abs(c.x - CENTER.x) + abs(c.y - CENTER.y) > max_depth:
                pruned.append(c)
        expansion_frontier = pruned
        added += 1
    if added < expansion_count:
        push_warning("DungeonGenerator: expansion only placed {a}/{e} rooms — seed={s}".format({
            "a": added, "e": expansion_count, "s": seed_id,
        }))
    print("[DungeonGenerator] expansion seed={s} max_depth={d} rooms_added={a}".format({
        "s": seed_id, "d": max_depth, "a": added,
    }))


func _get_expansion_neighbours(cell: Vector2i, occupied: Dictionary, min_depth: int) -> Array[Vector2i]:
    var result: Array[Vector2i] = []
    var offsets: Array[Vector2i] = [Vector2i(1, 0), Vector2i(-1, 0), Vector2i(0, 1), Vector2i(0, -1)]
    for offset: Vector2i in offsets:
        var neighbour: Vector2i = cell + offset
        if neighbour.x < 0 or neighbour.x >= GRID_SIZE or neighbour.y < 0 or neighbour.y >= GRID_SIZE:
            continue
        if occupied.has(neighbour):
            continue
        var depth: int = abs(neighbour.x - CENTER.x) + abs(neighbour.y - CENTER.y)
        if depth > min_depth:
            result.append(neighbour)
    return result
```
