# Implementation Plan: Alchemy Lab — Essence Condenser Upgrade

**Branch**: `045-shard-generator` | **Date**: 2026-03-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/045-shard-generator/spec.md`

## Summary

Add an "Essence Condenser" upgrade to the Alchemy Lab that passively generates shards over time. Three purchasable levels (costs 600 / 1200 / 2400 gold) yield 2 / 3 / 5 shards/hour respectively. Passive income ticks every frame; offline credit is applied on startup using the shared Gold Storage cap. All balance values live in `meta_config.json`; the implementation pattern mirrors the existing gold generator.

## Implementation Notes

### Call Order in MetaManager._ready()

`apply_offline_shards` **must** be called before `apply_offline_gold` because `apply_offline_gold` writes `gold_last_saved_timestamp` via `_save()`. Both need to read the original persisted timestamp.

### Shard Accumulator Pattern

`shard_accumulator: float` buffers sub-shard progress. Each tick:
- Add `delta × rate / 3600` to accumulator
- Drain `floori(accumulator)` as whole shards into `total_shards` (via `add_shards`)
- Subtract the drained integer from accumulator

This gives exact shard rates with no systematic drift.

### Offline Formula

`earned = floori(float(mini(elapsed, cap_seconds)) × rate / 3600.0)`

Same formula as gold offline credit. `cap_seconds` comes from `get_gold_storage_cap_seconds(base_hours, hours_per_level)` — no new config key needed.

### Cost Reuse

`get_upgrade_cost(shard_generator_level, 600, 2.0)` → 600 / 1200 / 2400. Uses the existing helper without modification.
