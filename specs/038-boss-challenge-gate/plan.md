# Implementation Plan: Boss Challenge Gate

**Branch**: `038-boss-challenge-gate` | **Date**: 2026-03-12 | **Spec**: [spec.md](spec.md)

## Summary

Gate the Boss Challenge Mode purchase behind the first endless-mode boss kill. On that kill, show a one-time popup (text from JSON) acknowledging the achievement and informing the player that Boss Challenge is now purchasable in the Mage Tower. The popup appears after any relic offer resolves. Until the kill, the Boss Challenge entry in the Mage Tower upgrade screen shows "Major essence required" and is non-interactive.

## Technical Context

**Language/Version**: GDScript (Godot 4.6)
**Primary Dependencies**: Godot engine autoloads — MetaManager, ResourceManager
**Storage**: `user://meta_save.json` — `first_boss_killed` flag already persisted; no new fields
**Testing**: Manual in-editor play testing
**Target Platform**: Android mobile portrait 1080×1920; Windows dev
**Performance Goals**: 60 fps — all new nodes are static Controls; no per-frame logic
**Constraints**: Mobile renderer; no Forward+ effects; `@export var` for all node refs
**Scale/Scope**: 1 new scene, 1 new script, 3 modified scripts, 1 modified JSON

## Constitution Check

- **I. Single Responsibility**: `BossKillPopup.gd` owns display and dismiss only. `Main.gd` coordinates post-boss flow timing (it already owns this concern). `MageTowerUpgradeScreen.gd` owns upgrade button display. No responsibility bleeds between them. ✅
- **II. Data-Driven Content**: Popup message text and gate text both read from `meta_config.json`. No string constants in `.gd` files. ✅
- **III. Mobile-First**: New nodes are `Control`, `Label`, `Button` — zero additional draw calls beyond existing hub UI. No per-frame logic. ✅
- **IV. Editor-Centric**: `BossKillPopup.tscn` created in Godot Editor. All child-node refs via `@export var`. No `$NodeName` literals. ✅
- **V. Simplicity & YAGNI**: No generic "notification" abstraction — `BossKillPopup` is a specific scene for a specific event. Gate logic is a prepended guard in the existing `_apply_entry` helper — no new abstraction. ✅
- **VI. Early Return**: `_apply_entry` gate check is a guard clause at the top of the function. All new methods exit at first unmet precondition. ✅

## Phase 0: Research ✅

See [research.md](research.md). All decisions resolved:

1. First-kill detection via `MetaManager.endless_boss_kill_count == 1` — no new signals or snapshot variables.
2. Popup inserted at `_show_boss_victory_overlay()` — single convergence point for both relic and no-relic paths.
3. Gate text in JSON under `mage_tower.upgrades.boss_challenge.gate_text`.
4. `_apply_entry` extended with opt-in `gate_prop` / `gate_text` guard — no effect on entries without these keys.
5. Popup scene at `scenes/ui/boss_kill_popup/` following existing CanvasLayer pattern.

## Phase 1: Implementation Steps

### Step 1 — Data layer (code only)

**`data/meta_config.json`**:
- Add `"popup_message": "<text>"` under `mage_tower.first_boss_killed` (new nested key)
- Add `"gate_text": "Major essence required"` to `mage_tower.upgrades.boss_challenge`

Resulting structure (additions only):
```json
"mage_tower": {
    "first_boss_killed": {
        "popup_message": "You have defeated the boss. Boss Challenge Mode can now be purchased in the Mage Tower."
    },
    "upgrades": {
        "boss_challenge": { "name": "Boss Challenge Mode", "cost": 200, "gate_text": "Major essence required" }
    }
}
```

### Step 2 — BossKillPopup script (code only)

**`scenes/ui/boss_kill_popup/BossKillPopup.gd`**:
- `class_name BossKillPopup extends Control`
- Signal: `ok_pressed`
- Exports: `_message_label: Label`, `_ok_button: Button`
- `setup(message: String)` — sets `_message_label.text = message`
- `_ready()` — connects `_ok_button.pressed → func() → ok_pressed.emit()`

### Step 3 — MageTowerUpgradeScreen (code only)

**`scenes/hub/MageTowerUpgradeScreen.gd`**:

Update `_entries` construction in `_ready()` — add `gate_prop` and `gate_text` to the boss_challenge entry:
```gdscript
upgrades.get("boss_challenge", {}).merged({
    "button": _bc_button,
    "owned_prop": "is_boss_run_unlocked",
    "purchase": MetaManager.purchase_boss_run,
    "gate_prop": "is_first_boss_killed",
    "gate_text": upgrades.get("boss_challenge", {}).get("gate_text", ""),
})
```

Update `_apply_entry(cfg: Dictionary)` — prepend gate guard:
```gdscript
func _apply_entry(cfg: Dictionary) -> void:
    var button: Button = cfg.get("button")
    var gate_prop: String = cfg.get("gate_prop", "")
    if gate_prop != "" and not MetaManager.get(gate_prop):
        button.text = cfg.get("gate_text", "")
        button.disabled = true
        return
    # existing owned/cost logic unchanged below
    ...
```

### Step 4 — Main.gd (code only)

**`scenes/core/Main.gd`**:

- Add fields: `_boss_kill_popup_layer: CanvasLayer`, `_first_boss_popup_pending: bool`
- Preload: `const _BOSS_KILL_POPUP_SCENE = preload("res://scenes/ui/boss_kill_popup/BossKillPopup.tscn")`
- In `_on_boss_room_cleared()`: after existing logic, add:
  ```gdscript
  if MetaManager.endless_boss_kill_count == 1:
      _first_boss_popup_pending = true
  ```
- Extract or update `_show_boss_victory_overlay()`:
  ```gdscript
  func _show_boss_victory_overlay() -> void:
      if _first_boss_popup_pending:
          _first_boss_popup_pending = false
          _show_boss_kill_popup()
          return
      # existing overlay instantiation unchanged
      ...
  ```
- Add `_show_boss_kill_popup()`:
  ```gdscript
  func _show_boss_kill_popup() -> void:
      var message: String = ResourceManager.get_meta_config().get("mage_tower", {}).get("first_boss_killed", {}).get("popup_message", "")
      _boss_kill_popup_layer = CanvasLayer.new()
      add_child(_boss_kill_popup_layer)
      var popup: BossKillPopup = _BOSS_KILL_POPUP_SCENE.instantiate()
      _boss_kill_popup_layer.add_child(popup)
      popup.setup(message)
      popup.ok_pressed.connect(_on_boss_kill_popup_ok)
  ```
- Add `_on_boss_kill_popup_ok()`:
  ```gdscript
  func _on_boss_kill_popup_ok() -> void:
      _boss_kill_popup_layer.queue_free()
      _boss_kill_popup_layer = null
      _show_boss_victory_overlay()
  ```
- In `_on_run_started()` and `_on_run_ended()`: free `_boss_kill_popup_layer` if non-null (same guard pattern as `_boss_victory_layer`).

### Step 6 — Validation

- Open Mage Tower upgrade screen before any boss kill: Boss Challenge shows "Major essence required", disabled.
- Add shards via DevPanel: Boss Challenge still shows "Major essence required" (shard balance irrelevant).
- Kill boss in endless run (first time): popup appears with message text before victory overlay.
- Kill boss with relic system active: relic offer appears first, then popup after relic pick, then victory overlay.
- Tap OK on popup: victory overlay appears.
- Return to hub, open Mage Tower: Boss Challenge now shows normal cost and affordability.
- Kill boss a second time: no popup.
- Restart game after first kill: Boss Challenge still shows normal cost (flag persisted).
- Dungeon Expansion and Relic System entries unchanged throughout.
