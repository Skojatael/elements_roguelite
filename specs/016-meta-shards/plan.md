# Implementation Plan: Meta Currency — Shards

**Branch**: `016-meta-shards` | **Date**: 2026-03-01 | **Spec**: [spec.md](spec.md)

## Summary

Add a persistent meta currency called "shards". At the end of every run, `essence_cashed_out` (from `RunSummary`) is converted 1:1 into shards (floored integer). MetaManager computes the amount, adds it to `MetaState.total_shards`, and SaveManager persists the updated total to `user://meta_save.json`. MetaManager and SaveManager (currently stub autoloads) receive their first real implementations in this feature.

## Technical Context

**Language/Version**: GDScript (Godot 4.6), static typing throughout
**Primary Dependencies**: Godot built-ins (`FileAccess`, `JSON`); existing autoloads `RunManager`, `ResourceManager`
**Storage**: `user://meta_save.json` — JSON file via `FileAccess`
**Testing**: Manual validation scenarios in `quickstart.md`
**Target Platform**: Android mobile (portrait); Windows for development
**Project Type**: Mobile game (single project)
**Performance Goals**: File I/O only at run end — no per-frame cost
**Constraints**: All balance values in JSON; no hardcoded constants in GDScript; `floori()` for all currency conversions

## Implementation Notes

### MetaState
- `class_name MetaState extends RefCounted`
- Single field: `var total_shards: int = 0`
- Parallel to `RunState`, `PlayerState`, `RunSummary` in `scripts/data_models/`

### SaveManager
- Save path constant: `const SAVE_PATH = "user://meta_save.json"`
- `save_meta_state(state: MetaState) -> void` — serializes to `{"total_shards": N}`, writes via `FileAccess`
- `load_meta_state() -> MetaState` — reads file if present; returns `MetaState.new()` (total=0) on missing/malformed file; never returns null
- Remove the empty `_process()` stub

### ResourceManager
- `get_meta_config() -> Dictionary` — reads and caches `data/meta_config.json`; follows existing `get_dungeon_config()` pattern

### MetaManager
- `var meta_state: MetaState` — loaded from SaveManager in `_ready()`; non-null at all times
- Connect `RunManager.run_ended` in `_ready()` using a lambda (arg mismatch pattern: signal passes `EndReason`, handler needs no arg OR uses it)
- `_on_run_ended(reason: RunManager.EndReason)` — reads `RunManager.run_summary.essence_cashed_out`, computes `floori(essence × rate)`, adds to total, saves, prints log
- Remove empty `_process()` stub

### data/meta_config.json
```json
{
  "shard_conversion_rate": 1.0
}
```
