# Implementation Plan: Mage Tower

**Branch**: `037-mage-tower` | **Date**: 2026-03-09 | **Spec**: [spec.md](spec.md)

## Summary

Add a "Mage Tower" hub building with ruined and restored visual states (mirroring the Magic Forge pattern). Once restored for 200 shards, the tower opens a system upgrades screen where players can individually purchase three system unlocks: Dungeon Expansion (200 shards), Relic System (100 shards), and Boss Challenge Mode (200 shards). The existing `AdventuringGearShop` and `BossRunShop` hub nodes are removed; the relic auto-unlock mechanic is removed. All unlock logic migrates into the Mage Tower purchase flow.

## Technical Context

**Language/Version**: GDScript (Godot 4.6)
**Primary Dependencies**: Godot engine autoloads — MetaManager, ResourceManager, SaveManager
**Storage**: `user://meta_save.json` (new field: `mage_tower_unlocked`)
**Testing**: Manual in-editor play testing
**Target Platform**: Android mobile portrait 1080×1920; Windows dev
**Performance Goals**: 60 fps — all UI elements are static Controls; no per-frame logic
**Constraints**: Mobile renderer; no Forward+ effects; `@export var` for all node refs
**Scale/Scope**: 3 new scenes, 3 new scripts, 4 modified scripts, 1 modified JSON

## Constitution Check

- **I. Single Responsibility**: MageTower.gd owns zone interaction only. RestoreTowerOverlay.gd owns the restore dialog only. MageTowerUpgradeScreen.gd owns the upgrades list only. MetaManagerImpl gains two new purchase methods in the same meta-purchase domain — no SRP violation. ✅
- **II. Data-Driven Content**: All four shard costs read from `data/meta_config.json`. No numeric constants in `.gd` files. ✅
- **III. Mobile-First**: All new nodes are `Control`/`ColorRect`/`Button`/`Label` — zero draw-call overhead beyond existing hub UI. ✅
- **IV. Editor-Centric**: All scenes created in Godot Editor. Every child-node reference via `@export var`. No `$NodeName` literals. ✅
- **V. Simplicity & YAGNI**: Three unlock rows are three independent `@export var` groups — no generic abstraction introduced (only one screen uses them). Removed dead code (AdventuringGearShop, BossRunShop, auto-unlock paths). ✅
- **VI. Early Return**: All new methods guard at the earliest precondition failure. ✅

## Phase 0: Research ✅

See [research.md](research.md). All decisions resolved:

1. Relic auto-unlock paths removed entirely (no existing saves).
2. No kill-count prerequisites for individual system unlocks.
3. AdventuringGearShop and BossRunShop nodes + scripts deleted.
4. Three static `@export var` groups in upgrade screen (no generic row abstraction).
5. Mage Tower positioned at approximately `(−400, 200)` hub-local (tuned in Editor).
6. Dungeon Expansion + Boss Challenge reuse existing purchase methods; Relic System gets new method.

**`scripts/data_models/MetaState.gd`**:
- Add `var mage_tower_unlocked: bool = false`

**`scripts/managers/SaveManager.gd`** (SaveManagerImpl):
- Add `"mage_tower_unlocked": state.mage_tower_unlocked` to `save_meta_state()` dict
- Add `state.mage_tower_unlocked = bool(parsed.get("mage_tower_unlocked", false))` to `load_meta_state()`

### Step 2 — MetaManagerImpl (code only)

**`scripts/managers/MetaManager.gd`** (MetaManagerImpl):
- Delete `try_activate_relic_offers()` method
- Delete `unlock_adventurer_bag()` method
- Add `purchase_mage_tower(cost: int, save_manager: Node) -> bool`:
  ```gdscript
  if meta_state.mage_tower_unlocked:
      return false
  if not can_spend(cost):
      return false
  meta_state.total_shards -= cost
  meta_state.mage_tower_unlocked = true
  save_manager.save_meta_state(meta_state)
  return true
  ```
- Add `purchase_mage_tower_relic_system(cost: int, save_manager: Node) -> bool`:
  ```gdscript
  if meta_state.relic_offers_active:
      return false
  if not can_spend(cost):
      return false
  meta_state.total_shards -= cost
  meta_state.adventurer_bag_unlocked = true
  meta_state.relic_offers_active = true
  save_manager.save_meta_state(meta_state)
  return true
  ```

### Step 3 — MetaManager autoload (code only)

**`autoload/MetaManager.gd`**:
- Remove call to `_impl.try_activate_relic_offers(SaveManager)` from `_on_hub_entered()`; remove `_on_hub_entered()` if that was its sole purpose
- Remove `_impl.unlock_adventurer_bag(SaveManager)` call from `_on_room_cleared()` elite branch; clean up surrounding elite detection logic if it was solely serving that call
- Add computed property:
  ```gdscript

  ```

- Add `purchase_mage_tower_relic_system() -> bool` (reads `mage_tower_relic_system_cost`, delegates, emits `shards_changed`)

### Step 4 — New scripts (code only, before scene creation)

**`scenes/hub/RestoreTowerOverlay.gd`**:
- `class_name RestoreTowerOverlay extends Control`
- Signals: `restore_pressed`, `maybe_later_pressed`
- `@export var _restore_button: Button`, `@export var _later_button: Button`
- `_ready()`: reads `mage_tower_cost` from config; sets button text; disables restore button if `not MetaManager.can_spend(cost)`; connects signals

**`scenes/hub/MageTower.gd`**:
- `class_name MageTower extends Control`
- `@export var _ruined_visual: ColorRect`, `@export var _magic_visual: ColorRect`
- `@export var _label: Label`, `@export var _button: Button`
- `@export var _restore_overlay_scene: PackedScene`, `@export var _upgrade_screen_scene: PackedScene`
- `var _overlay_layer: CanvasLayer = null`
- `_ready()`: connect button, `MetaManager.shards_changed`, `GlobalSignals.hub_entered` → `_update_visuals()`
- `_update_visuals()`: toggle ruined/magic visuals based on `MetaManager.is_mage_tower_unlocked`
- `_on_tower_pressed()`: guard `_overlay_layer != null`; branch on `is_mage_tower_unlocked`
- `_show_restore_overlay()`, `_show_upgrade_screen()`, `_close_overlay()`, `_on_restore_pressed()` — mirrors MagicForge pattern exactly

**`scenes/hub/MageTowerUpgradeScreen.gd`**:
- `class_name MageTowerUpgradeScreen extends Control`
- Signal: `close_pressed`
- Nine exports: `_de_button`, `_de_unlocked_label`, `_rs_button`, `_rs_unlocked_label`, `_bc_button`, `_bc_unlocked_label`, `_close_button`
- `_ready()`: connect all buttons; connect `MetaManager.shards_changed` → `_update_entries()`; call `_update_entries()`
- `_update_entries()`: reads three costs from config; for each system sets button text, disabled state, and shows button vs unlocked label based on ownership flag
- `_on_de_buy()`: calls `MetaManager.purchase_adventuring_gear()`, then `_update_entries()`
- `_on_rs_buy()`: calls `MetaManager.purchase_mage_tower_relic_system()`, then `_update_entries()`
- `_on_bc_buy()`: calls `MetaManager.purchase_boss_run()`, then `_update_entries()`
