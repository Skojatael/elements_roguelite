# Implementation Plan: Magic Missile Burn

**Branch**: `054-magic-missile-burn` | **Date**: 2026-03-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/054-magic-missile-burn/spec.md`

## Summary

Add an uncommon relic (`burn`, display name "Living Ember") that causes magic missile hits to apply a burn status effect: 10% of projectile damage per tick, ticking once per second, base duration 2s. Re-hitting extends the duration by 2s per hit. All parameters are data-driven in `data/skills.json`. Burn state is managed by a new pure-logic `BurnEffect` RefCounted class, processed inside `Enemy._physics_process()`. Kill credit from burn ticks flows through the existing `take_damage()` ‚Üí `StatsComponent.died` ‚Üí `Enemy.defeated` ‚Üí `RunManager.enemies_slain` chain automatically.

```json
  "tags": ["projectile", "burn"],

  "description": "Magic Missile ignites enemies, dealing 20% attack damage over 2s. Additional hits extend the burn."
}
```

### Step 3 ‚Äî New: `scripts/data_models/BurnEffect.gd`

Pure `RefCounted` with fields `remaining_duration`, `tick_damage`, `_seconds_until_next_tick`. Methods: `apply()`, `extend()`, `process()`, `is_active()`. See contracts for exact signatures.

`process(delta)`:
1. If `not is_active()`: return `0.0`.
2. Decrement `remaining_duration` by `delta`; clamp to 0.
3. Decrement `_seconds_until_next_tick` by `delta`.
4. If `_seconds_until_next_tick <= 0.0` and remaining was > 0 before decrement: reset `_seconds_until_next_tick += 1.0`, return `tick_damage`.
5. Return `0.0`.

### Step 4 ‚Äî `RelicManagerImpl.gd`

Add after `has_chain_relic()`:
```gdscript
func has_burn_relic() -> bool:
    return active_relic_ids.has("burn")
```

### Step 5 ‚Äî `RelicManager.gd`

Add after `has_chain_relic()`:
```gdscript
func has_burn_relic() -> bool:
    return _impl.has_burn_relic()
```

### Step 6 ‚Äî `Enemy.gd`

1. Add field: `var _burn: BurnEffect = null`
2. Add `on_burn_hit(tick_dmg, base_duration, extend_seconds)` ‚Äî see contracts.
3. Add to `_physics_process(delta)` **before** the contact damage block:
   ```gdscript
   if _burn != null and _burn.is_active():
       var burn_dmg: float = _burn.process(delta)
       if burn_dmg > 0.0:
           take_damage(burn_dmg)
   ```

### Step 7 ‚Äî `SkillComponent.gd`

1. Add three `var _burn_*` fields.
2. Read from JSON in `_load_skill_data()`.
3. Pass to `projectile.setup()`.

### Step 8 ‚Äî `Projectile.gd`

1. Add three `var _burn_*` fields.
2. Extend `setup()` to accept and store them.
3. In `_on_body_entered()` after `primary.take_damage(_damage)`:
   ```gdscript
   if RelicManager.has_burn_relic():
       primary.on_burn_hit(_damage * _burn_damage_per_tick, _burn_duration, _burn_extend_seconds)
   ```
4. In `_try_chain()` after `chain_target.take_damage(...)`:
   ```gdscript
   if RelicManager.has_burn_relic():
       chain_target.on_burn_hit(_damage * _burn_damage_per_tick, _burn_duration, _burn_extend_seconds)
   ```

### Step 9 ‚Äî `tests/unit/test_burn_effect.gd`

Unit tests covering (all pure ‚Äî no autoloads):
- `apply()` sets state correctly
- First tick fires at t=1.0s, not at t=0.9s
- Second tick fires at t=2.0s
- `is_active()` returns false after duration expires
- `extend()` adds to remaining duration
- Re-hit on active burn: extend, no new apply
- Re-hit on expired burn: fresh apply
- `process()` returns 0.0 when inactive
